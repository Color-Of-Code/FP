# SysML local VS Code extension

A self-contained VS Code extension for the SysML subset used by this repo:

- syntax highlighting (TextMate grammar generated by `langium-cli`)
- live parse-error diagnostics via a Langium language server

Nothing here is published to a marketplace — the extension runs out of this folder.

## Build

```sh
make -C .tools vscode-ext
```

That installs deps in this folder and runs `tsc`. Output ends up under `dist/`.

## Install (one of two options)

### Option A — symlink into the user extensions folder (no `.vsix`)

```sh
make -C .tools vscode-ext-link
```

This symlinks `.tools/vscode-sysml/` to `~/.vscode/extensions/fp-local.sysml-fp-local-0.0.1/`.
Restart VS Code (or run _Developer: Reload Window_). `.sysml` files now have highlighting, brackets,
and live diagnostics.

To uninstall, delete the symlink:

```sh
rm ~/.vscode/extensions/fp-local.sysml-fp-local-0.0.1
```

### Option B — package and install a `.vsix`

Requires `@vscode/vsce` (`pnpm dlx @vscode/vsce package` works without a global install):

```sh
cd .tools/vscode-sysml
pnpm dlx @vscode/vsce package    # produces sysml-fp-local-0.0.1.vsix
code --install-extension sysml-fp-local-0.0.1.vsix
```

The `.vsix` is just a zip of this folder; nothing is uploaded.

## After grammar changes

Whenever you edit `.tools/sysml/sysml.langium`:

```sh
make -C .tools langium-gen   # refresh sysml/generated/
make -C .tools vscode-ext    # rebuild server with new grammar
```

Then reload the VS Code window.
